# API Design

Base URL: `http://localhost:8000/api/v1`

Interactive docs available at `http://localhost:8000/docs` (Swagger UI, auto-generated by FastAPI).

---

## Error Envelope

All 4xx and 5xx responses return:

```json
{
  "detail": "Human-readable message",
  "code": "machine_readable_code",
  "fields": null
}
```

`fields` is a map of `{ field_name: [error_message] }` for validation errors (422), otherwise `null`.

## Endpoints

### `GET /health`

Liveness probe.

**Response 200**
```json
{ "status": "ok" }
```

### `GET /weather/current`

Ad-hoc weather lookup. **Does not persist to database.**

**Query params**

| Param | Type | Required | Notes |
|---|---|---|---|
| `location` | string | one of `location` or `lat`+`lon` | Free-text: city, zip, landmark, GPS string |
| `lat` | float | one of `location` or `lat`+`lon` | Decimal degrees |
| `lon` | float | one of `location` or `lat`+`lon` | Decimal degrees |

**Response 200**
```json
{
  "resolved_location": "Tokyo, Tokyo, Japan",
  "latitude": 35.68950,
  "longitude": 139.69171,
  "current": {
    "temperature": 28.4,
    "feels_like": 31.2,
    "humidity": 72,
    "wind_speed": 14.4,
    "condition": "Partly cloudy",
    "condition_code": 2,
    "aqi": 42,
    "aqi_label": "Good"
  },
  "forecast": [
    {
      "date": "2026-06-25",
      "temp_min": 23.1,
      "temp_max": 31.8,
      "condition": "Overcast",
      "condition_code": 3,
      "precipitation_sum": 0.0,
      "wind_speed_max": 18.0
    }
  ]
}
```

**Errors**
- `404 location_not_found` — Geocoder returned no results.
- `502 upstream_error` — Open-Meteo unreachable.

### `POST /weather`

CREATE a saved weather search with a date range. Geocodes the location, fetches weather for `start_date`, and persists the record.

**Request body**
```json
{
  "location_query": "Paris",
  "start_date": "2026-07-01",
  "end_date": "2026-07-05"
}
```

**Validation rules**
- `location_query`: 1–255 chars, non-blank.
- `start_date`: must be a valid date, not more than 2 years in the past.
- `end_date`: >= `start_date`, <= `start_date + 16 days` (Open-Meteo forecast horizon).
- Location must resolve via geocoder; ambiguous input is disambiguated to best match.

**Response 201**
```json
{
  "id": 1,
  "location_query": "Paris",
  "resolved_location_name": "Paris, Île-de-France, France",
  "latitude": 48.85341,
  "longitude": 2.34880,
  "start_date": "2026-07-01",
  "end_date": "2026-07-05",
  "weather_condition": "Clear sky",
  "temperature": 22.3,
  "humidity": 55,
  "wind_speed": 11.2,
  "created_at": "2026-06-24T10:00:00Z",
  "updated_at": "2026-06-24T10:00:00Z"
}
```

**Errors**
- `404 location_not_found`
- `422 validation_error` — invalid dates or blank query.

### `GET /weather`

READ all saved weather searches (paginated).

**Query params**

| Param | Default | Notes |
|---|---|---|
| `limit` | 20 | Max 100 |
| `offset` | 0 | |

**Response 200**
```json
{
  "total": 42,
  "items": [ { ...WeatherSearch... } ]
}
```

---

### `GET /weather/{id}`

READ one record by ID.

**Response 200** — single `WeatherSearch` object (same shape as POST 201).

**Errors**
- `404 not_found`

---

### `PUT /weather/{id}`

UPDATE an existing record. Only `location_query`, `start_date`, and `end_date` are user-editable. Weather snapshot fields are re-fetched server-side on update.

**Request body** (all fields optional)
```json
{
  "location_query": "Lyon",
  "start_date": "2026-07-10",
  "end_date": "2026-07-12"
}
```

Same validation rules as POST apply to any provided fields.

**Response 200** — updated `WeatherSearch` object.

**Errors**
- `404 not_found`
- `404 location_not_found`
- `422 validation_error`

---

### `DELETE /weather/{id}`

DELETE a record permanently.

**Response 204** — no body.

**Errors**
- `404 not_found`

---

### `GET /weather/export`

Export all saved records.

**Query params**

| Param | Values | Default |
|---|---|---|
| `format` | `csv`, `json`, `md` | `csv` |

**Response**

Content-Type is set per format:
- `csv` → `text/csv; charset=utf-8`; `Content-Disposition: attachment; filename="weather_searches.csv"`
- `json` → `application/json`
- `md` → `text/markdown`

### `POST /advice`

Generate AI travel advice from current weather and AQI data using NVIDIA NIM.

**Request body**
```json
{
  "location": "Tokyo, Japan",
  "current": {
    "temperature": 28.4,
    "humidity": 72,
    "wind_speed": 14.4,
    "condition": "Partly cloudy",
    "aqi": 42,
    "aqi_label": "Good"
  },
  "forecast": [
    { "date": "2026-06-25", "temp_min": 23.1, "temp_max": 31.8, "condition": "Overcast", "precipitation_sum": 0.0 }
  ]
}
```

**Response 200**
```json
{
  "clothing": ["Light breathable layers", "Rain jacket for overcast days"],
  "packing": ["Sunscreen SPF 50+", "Portable fan", "Umbrella"],
  "activities": ["Morning temple visits before peak heat", "Indoor museums during afternoon"],
  "travel_considerations": ["Humidity makes 28°C feel closer to 32°C", "Good air quality — outdoor activities are safe"],
  "warnings": []
}
```

**Errors**
- `503 ai_unavailable` — NVIDIA NIM key missing or API unreachable (graceful fallback, not a crash).

### `GET /integrations/aqi`

Fetch Air Quality Index for a coordinate pair (used internally by `/weather/current`, exposed for direct calls).

**Query params**: `lat` (float, required), `lon` (float, required).

**Response 200**
```json
{
  "aqi": 42,
  "label": "Good",
  "pm2_5": 6.2,
  "pm10": 11.4
}
```
